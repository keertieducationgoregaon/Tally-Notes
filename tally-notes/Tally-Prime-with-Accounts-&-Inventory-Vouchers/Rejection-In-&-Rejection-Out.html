1. Introduction

Rejection Notes are used when goods are returned before final Purchase/Sales invoice is recorded.

They update physical stock only — no accounting effect.

2. Concept of Rejections

Rejections occur when goods are returned due to damage, wrong item, excess supply, or customer dissatisfaction — before billing.

3. Rejection In – Customer Returns Goods to Us

Effect: Stock ↑ | Delivery Note pending ↓ | No accounting entry

Shortcut: Alt + F6

4. Rejection Out – We Return Goods to Supplier

Effect: Stock ↓ | Receipt Note pending ↓ | No accounting entry

Shortcut: Alt + F7

4. Voucher Activation

  1. Option 1
  2. Gateway of Tally > Masters > Alter > Voucher Type
  3. > Rejection In > Activate this Voucher Type > Yes
  1. Option 2
  2. Gateway of Tally > Transactions > Voucher > F10/Other Voucher
  3. > Show Inactive > Rejection In > Activate this Voucher > Yes

Same For Rejection Out

6. Rejection In Voucher – Steps

  1. Vouchers → Alt + F6
  2. Select Customer
  3. Select related Delivery Note (recommended)
  4. Choose returned item
  5. Enter quantity & reason
  6. Save

7. Rejection Out Voucher – Steps

  1. Vouchers → Alt + F7
  2. Select Supplier
  3. Select related Receipt Note
  4. Choose item to return
  5. Enter quantity & reason
  6. Save

SALES SIDE
SO → Delivery Note → Rejection In → Sales Voucher

11. Common Mistakes

  • Confusing Rejection Notes with Debit/Credit Notes
  • Not linking DN/RN
  • Using Rejection In for supplier returns
  • Wrong quantity causing stock mismatch
  • Forgetting to enable Rejection Notes in F11

12. Revision Summary

  • Rejection In → Customer returns → Stock ↑
  • Rejection Out → We return to supplier → Stock ↓
  • No accounting effect
  • Always link to DN (for RI) or RN (for RO)
  • Used before final invoice
